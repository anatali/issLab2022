Boundary walker (boundarywalker.qak)

We assume that:
  • the robot works in an empty, rectangular room limited by four walls;
  • along each border of the room there is a rectangular area in which the robot can freely move.

Mapping walker (mappingwalker.qak)

We assume that
  • the robot works in an empty, rectangular room with a obstacle-free area along the borders
  • that a map of the room is already available
The robot:
  1. loads the boundary-free-room map (e.g. roomBoundary.txt);
  2. tries to reach each not-yet-explored cell of the room;
  3. when finds an obstacle during its trip, it attempts to reach another not-yet-explored cell;
  4. when all the cells have been explored, it returns to its home.

An approach for the butler project

Architecture
We start from LAB14 | Resource(domain) model.
The starting system can be composed of the following actors:
  1. a basic robot that actuates move comands
  2. a resourcemodel that represents the (CoAP accessible) core of the system
  3. a robot mind that reacts to changes in the resourcemodel and to events of the world
  4. a sonarhandler that looks at the sonar on the robot
  5. a onestepahead that reacts to onestep requests by moving the robot ahead for the time given in the request message. It sends to the caller the answer (dispatch) stepOk if the move is completed with success or stepFail if the move has found an obstacle.
and a robot mind robotmind.qak

Planning

  • setGoal(actor:ActorBasic, x: String, y: String)
    Sets the planning goal to the cell (x,y).

  • doPlannedMove(actor:ActorBasic, move:String)
    Updates the room map with the given move.

  • doPlan(actor:ActorBasic)
    Builds a plan for the current goal and stores the moves in the actor kb as a sequence of move/1 facts.

Moving

  • moveAhead(actor:ActorBasic,stepTime:Int,pauseTime:Int=PauseTime,dest:String="resourcemodel")
    Moves the robtot along the current direction for the given stepTime and updates the room map. Afterwards, it waits for the given pauseTime.

  • rotate(actor:ActorBasic,move:String,pauseTime:Int=PauseTime)
    Rotates the robtot and updates the room map. Afterwards, it waits for the given pauseTime.

  • attemptTomoveAhead(actor:ActorBasic,stepTime:Int,dest:String="resourcemodel")
    Moves the robot ahead (w) by sending to dest the 'request' oneStep with the given stepTime. The requested actor will answer with the dispatch stepOk or stepFail.

  • updateMapAfterAheadOk(actor:ActorBasic)
    Updates the room map (after receiving a stepOk) with the move w .

Utilities

  • showCurrentRobotState( )
    Prints the current position of the robot, its direction and the size of the room.

      Global architecture

      The global architecture can be summarized as follows:

      • The robot can be moved by sending a modelChange : modelChange( TARGET, VALUE ) dispatch to the resourceModel. For example:
        Move the robot ahead at model level 
        forward resourcemodel -m modelChange : modelChange(robot,w)
        Move the robot ahead at Kotlin level 
        forward("modelChange", "modelChange(robot,w)" ,"resourcemodel" )
        Move the robot ahead at CoAP level 
        coapClient = CoapClient("coap://$serverAddr:5683/resourcemodel)
coapClient.put("w", MediaTypeRegistry.TEXT_PLAIN)

      Essential architecture layers
    • Low-level layers can execute commands sent by higher-level layers.

    • A command changes the resource model. This change is perceived by the robotmind that has the responsibility to execute the command.

    • Low-level layers generate events that can be sensed and handled by the robotmind or by higher-level layers.

    • Layers separated by a continuous line can be executed on different contexts (nodes of a dsitributed system).
